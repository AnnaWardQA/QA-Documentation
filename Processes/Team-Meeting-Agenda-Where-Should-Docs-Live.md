# 📋 Where Should QA Documentation Live? — Team Meeting Agenda Item

## Agenda Item — Where Should QA Documentation Live?

- Each team member currently has their own personal repository — docs are generated from work items assigned to that individual
- Anyone on the team can view each other's repositories, but only the owner adds to it
- Keeps things simple and avoids anyone accidentally overwriting someone else's work
- Copilot generates the documents automatically from Azure DevOps PBIs — so everyone would follow the same process
- Option to move to a single shared team repository later if we decide everything should be in one central place
- Recommendation — keep it individual for now while we're still getting the process up and running, and review once the team is up to speed
